\## 1. 项目概述



本项目旨在构建一个名为“一点灵光”的Web应用，为用户提供极简的视频创作体验。用户通过输入一句话创意、上传剧本或小说，系统即可自动生成时长1-5分钟、剧情连贯的短视频。



\*\*核心价值：\*\*

\- \*\*低门槛\*\*：一句话即可生成视频，无需专业技能。

\- \*\*高质量\*\*：借鉴好莱坞工业流程，通过AI智能体（编剧、分镜师、导演、制片人）协同工作，保证视频的叙事性和观赏性。

\- \*\*风格多样\*\*：支持短剧、广告、小红书、旅拍等多种风格。



\*\*MVP核心功能：\*\*

1\. 实现前端用户界面与后端服务的完整交互。

2\. 实现\*\*创意生成视频\*\*（一句话创意 → 完整视频）核心功能。

3\. 实现\*\*剧本生成视频\*\*（完整剧本/小说章节 → 完整视频）核心功能。

4\. 确保生成视频的角色一致性和基本叙事连贯性。

5\. 提供流畅的用户体验和基本的任务管理功能。

\## 2. 技术架构



本项目采用前后端分离架构。



\### 2.1 总体架构



\- \*\*前端\*\*：用户交互界面。负责收集用户输入，发起API请求，展示生成进度和最终视频。

\- \*\*后端\*\*：核心业务逻辑与调度中心。负责用户管理、任务队列、调用多智能体视频生成引擎、任务状态管理、视频存储与分发。

\- \*\*视频生成引擎\*\*：后端内部核心模块。基于多智能体协作架构（编剧Agent、分镜师Agent、导演Agent、制片人Agent），执行从文本到视频的完整生成流水线。



\### 2.2 后端核心模块说明



后端需实现以下核心模块：



| 模块名称 | 功能描述 |

| :--- | :--- |

| \*\*Agent PE 引擎\*\* | 实现“扩写+对齐”两阶段处理，将用户输入转化为结构化剧本和模型友好指令。 |

| \*\*编剧 Agent\*\* | 负责理解和扩展用户创意，生成包含角色卡、场景表、对白、情节结构的完整剧本。 |

| \*\*分镜师 Agent\*\* | 将剧本转化为镜头级的分镜表，定义镜头类型、运动方式、时长和关键帧描述。 |

| \*\*导演 Agent\*\* | 统筹视觉资产规划，调用技能提取引擎，协调图像生成与视频生成，执行一致性校验。 |

| \*\*制片人 Agent\*\* | 编排并行化的镜头生产流水线，执行多维质量评估，管理资源分配与异常处理，完成最终合成。 |

| \*\*技能提取引擎\*\* | 从高质量参考视频中提炼叙事结构、运镜模式等技能特征，存入知识库供复用。 |

| \*\*质量评估引擎\*\* | 对生成视频进行技术质量（画质、一致性）和叙事质量（剧情完整性、戏剧张力）的多维度评估。 |

\## 5. 核心流水线流程



后端Celery Worker执行一个生成任务时，必须按以下流程调用各智能体模块：



1\. \*\*接收任务参数\*\*：`task\_id`, `mode`, `content`, `style`, `target\_duration`。

2\. \*\*更新状态\*\*：`status = PROCESSING`, `progress = 5`, `current\_stage = "编剧Agent正在创作剧本..."`。

3\. \*\*调用编剧Agent\*\*：将`content`和`style`传入，调用LLM API，生成结构化剧本（JSON）。输出包含：角色卡、场景列表（每个场景包含环境描述、对白、情绪基调）。

4\. \*\*更新状态\*\*：`progress = 25`, `current\_stage = "分镜师Agent正在设计分镜..."`。

5\. \*\*调用分镜师Agent\*\*：将剧本JSON传入，调用LLM API，生成分镜表。输出为镜头列表，每个镜头包含：镜头类型、运动方式、预估时长、关键帧描述。

6\. \*\*更新状态\*\*：`progress = 45`, `current\_stage = "导演Agent正在规划视觉资产并生成画面..."`。

7\. \*\*调用导演Agent\*\*：

&#x20;  - 遍历分镜表，为每个镜头生成图像提示词。

&#x20;  - 调用图像生成API生成关键帧/参考帧。

&#x20;  - 执行一致性校验（同一角色在不同镜头中保持外观一致）。

&#x20;  - 调用视频生成API，将关键帧扩展为2-5秒的视频片段。

8\. \*\*更新状态\*\*：`progress = 80`, `current\_stage = "制片人Agent正在合成最终视频..."`。

9\. \*\*调用制片人Agent\*\*：

&#x20;  - 使用FFmpeg将所有视频片段按顺序拼接。

&#x20;  - 根据剧本情绪，添加背景音乐（可选，使用免费音效库）。

&#x20;  - 添加字幕（如有对白）。

&#x20;  - 输出最终MP4文件到`./workspace/<task\_id>/final.mp4`。

10\. \*\*更新状态\*\*：`progress = 95`, `current\_stage = "质量评估中..."`。

11\. \*\*质量评估\*\*：调用质量评估引擎，对最终视频进行技术质量和叙事质量的评分，记录到数据库。

12\. \*\*完成\*\*：将视频文件移动到最终存储位置，生成访问URL。更新数据库：`status = SUCCESS`, `progress = 100`, `video\_url = "..."`。



> 若上述任何一步失败，捕获异常，更新`status = FAILED`，并记录`error\_message`。
